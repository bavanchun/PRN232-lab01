# PRN232 Lab 1 — LMS REST API

**MSSV:** SE173473
**Stack:** ASP.NET Core 10, EF Core 10, SQL Server 2022, Docker

## Run

Prereq: Docker Desktop.

```bash
cd labs/lab1/SE173473
docker compose up --build
```

Wait ~60s for SQL Server boot + EF migrate + seed.

## Endpoints

- Swagger UI: <http://localhost:8080/swagger>
- Root redirects → Swagger
- Submission endpoint: <http://localhost:8080/api/v1/courses/1/enrollments?expand=student>

## Architecture

3-layer: **API → Services → Repositories**, with 4 model types: Entity / Business / Request / Response.

### Resources (all support GET/POST/PUT/PATCH/DELETE)

| Resource | URL |
|---|---|
| Semesters | `/api/v1/semesters` |
| Courses | `/api/v1/courses` |
| Subjects | `/api/v1/subjects` |
| Students | `/api/v1/students` |
| Enrollments | `/api/v1/enrollments` |

Versioned `/api/v1/...` is canonical; unversioned `/api/...` aliases to v1.

### Nested resources

| Verb | URL | Notes |
|---|---|---|
| GET | `/api/v1/semesters/{id}/courses` | Courses in semester |
| GET | `/api/v1/courses/{id}/enrollments` | Submission endpoint, supports `?expand=student` |
| GET | `/api/v1/students/{id}/enrollments` | Student's enrollments |

## List API capabilities

All list endpoints support:

| Query | Example |
|---|---|
| Search | `?search=nguyen` |
| Sort | `?sort=fullName,-dateOfBirth` |
| Page | `?page=1&size=10` |
| Field selection | `?fields=studentId,fullName,email` |
| Expand | `?expand=student,course` |

Combined:
```
/api/v1/enrollments?search=Active&sort=-enrollDate&page=1&size=20&fields=enrollmentId,status&expand=student,course
```

## Response envelope

```json
{
  "success": true,
  "message": "Request processed successfully",
  "data": { ... },
  "errors": null,
  "pagination": { "page": 1, "pageSize": 10, "totalItems": 100, "totalPages": 10 },
  "_links": { "self": { "href": "/api/v1/students?page=1&size=10", "method": "GET" }, ... }
}
```

HTTP status codes: 200 / 201 / 204 / 400 / 404 / 500.

## HATEOAS

Every Response includes `_links` with: `self`, `update`, `patch`, `delete`, and related resources (e.g. `enrollments` on Student). Collection responses include pagination links: `first`, `prev`, `next`, `last`.

